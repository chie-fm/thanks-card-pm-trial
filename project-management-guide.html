Project Management Guide · 2025

管理ツール
導入ガイド

GitHub Copilot を軸とした開発チームのための、3つのプロジェクト管理ツール比較と導入手順。スクラム・カンバンどちらにも対応。

進め方の戦略
PHASE 01
スクラム
スプリント単位で開発。
イベント（計画・レビュー等）を定期実施
PHASE 02
カンバン
兼任が増えたら移行。
WIP制限でフロー管理
PHASE 03
柔軟対応
さらに負荷が高まれば
非同期・非定型に切り替え
⊙

GitHub Projects

GitHub ネイティブの統合型プロジェクト管理。Issue・PR・Copilot と同一エコシステムで完結。

🥇 推奨 · 無料

導入ステップ

1
リポジトリにProjectsを作成 GitHub リポジトリ → "Projects" タブ → "New project" → テンプレートから Scrum または Kanban を選択
2
IssueをProjectに紐付け Issue 作成時に右サイドバーの "Projects" からアサイン。ラベルで Story / Task / Bug を分類する
3
スプリントフィールドを追加 "+ Add field" → "Iteration" でスプリント期間を設定（例：2週間）。バックログから対象Issueをアサイン
4
Copilot と連携 Issue に要件を書き込み → Copilot Chat で #issue-番号 を参照して実装指示を出す
5
カンバンへの移行 "View" を追加 → "Board" レイアウトに切り替えるだけ。データはそのまま引き継がれる

Copilot 連携のポイント

Issue のタイトル・本文を構造化して書く（背景 / 受け入れ条件 / タスク）と Copilot が参照しやすい
PR を作成すると Copilot がコードレビューを自動実行し、該当 Issue に結果を紐付けられる
GitHub Actions でスプリント終了時に未完了 Issue を自動で次スプリントへ移動するワークフローを組める
Codespaces から直接 Issue を開き、ブランチ作成～コミット～PR まで一気通貫で完結する

向いているケース

追加コストゼロ GitHub メイン開発 小規模スタート 段階的スクラム
◈

Linear

開発者ファーストの高速プロジェクト管理。スプリント（Cycles）とカンバンを柔軟に切り替え可能。

🥈 次点 · 無料〜

導入ステップ

1
アカウント作成・ワークスペース設定 linear.app にアクセス → "Start for free" → チーム名・メンバーを追加（無料は最大メンバー数制限あり）
2
GitHub 連携を有効化 Settings → Integrations → GitHub → リポジトリを選択。PR 作成時に Issue が自動で "In Progress" へ移動する
3
Cycles（スプリント）を設定 Team Settings → Cycles → ON にして期間を設定。Issues をドラッグして Current Cycle にアサイン
4
カンバンへの移行 Cycles を OFF → "Board" ビューに切り替え。WIP 制限をステータスごとに設定してフロー管理
5
Copilot との連携 Linear Issue URL を Copilot Chat に貼り付けると内容を参照可能。PR タイトルに LIN-123 を含めると自動紐付け

Linear ならではの強み

キーボードショートカットが充実しており、Issue 作成・移動がマウスなしで完結。開発中に素早く記録できる
バーンダウンチャート・ベロシティ・サイクルタイムが自動で計算され、スクラムイベントに使いやすい
Slack 連携が優秀で、Issue の更新を特定チャンネルに通知でき、非同期コミュニケーションと相性が良い
Sub-issues でタスクの階層構造を作れるため、エピック → ストーリー → タスクの粒度管理がしやすい

向いているケース

本格スクラム管理 速度重視 メトリクス確認 Slack 連携
◻

Notion

ドキュメント・仕様書・タスクを一元管理。設計書・議事録・タスクをひとつの場所に集約できる。

🥉 予備案 · 無料〜

導入ステップ

1
ワークスペース作成 notion.so → "Get started free" → チームワークスペースを作成。テンプレートギャラリーから "Engineering Wiki" を使うと楽
2
スプリントデータベースを構築 新規ページ → "Database" → "Board" ビューを選択。ステータス（Backlog / Todo / In Progress / Done）を設定
3
GitHub 連携（GitHub Embed） Notion ページ内で /github と入力 → リポジトリの PR・Issue をブロックとして埋め込む
4
仕様書とタスクをリンク "Relation" プロパティを使い、仕様書ページとタスクカードを双方向リンク。Copilot の参照ソースにもなる
5
カンバンへの切り替え 既存データベースのビューを "Board" に追加するだけ。スプリントフィールドをフィルターして表示を切り替える

Notion ならではの強み

設計書・API 仕様・議事録・タスクが一箇所にまとまり、Copilot に渡すコンテキストとして活用できる
Notion AI を使えばタスクの説明文を自動生成したり、議事録からアクションアイテムを抽出できる
フリープランでも基本機能が使える。チーム全員が同じページを編集できるリアルタイムコラボが便利
プロジェクト初期の要件定義フェーズは Notion → 開発フェーズは GitHub Projects という使い分けも有効

向いているケース

ドキュメント重視 仕様書管理 議事録一元化 要件定義フェーズ

Git コマンド手順

$ mkdir my-project && cd my-project
# プロジェクトフォルダを作成
$ git init
# Git リポジトリを初期化
$ cp /path/to/project-management-guide.html .
# ダウンロードしたHTMLをフォルダにコピー
$ git add project-management-guide.html
$ git commit -m "docs: add project management tool guide"
# GitHub でリポジトリを新規作成してから以下を実行
$ git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git
$ git branch -M main
$ git push -u origin main
# GitHub Pages で公開する場合は Settings → Pages → main ブランチを選択
